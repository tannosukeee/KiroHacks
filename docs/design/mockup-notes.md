# Mockup Notes

These mockups define the current target look for Vybe Tutor's webview UI.

## Files

- `mockups/vybe-explain-panel.png`
- `mockups/vybe-calibration-screen.png`

## Main visual conclusions

The design should feel like a dark, focused, terminal-inspired learning layer. It should not feel like a generic chatbot or a bright classroom dashboard.

## Live explanation panel

Important details to preserve:

- Header label: `VYBE EXPLAIN`
- Amber `LIVE` badge
- Large monospace explanation text
- Inline code chips with amber text
- Line-based explanation title, such as `Line 3 · @cache decorator`
- Concise concept explanation
- Calibrated difficulty label, such as `Calibrated for: Advanced`
- `QUICK CHECK · +10 XP` section label
- Large visual prompt card
- Full-width answer choice buttons
- Dark background with muted borders

## Calibration screen

Important details to preserve:

- Dark full-screen onboarding surface
- Small uppercase progress label
- Large question heading
- Muted helper text
- Thin amber progress bar
- Large dark option cards
- Minimal Back/Next controls
- Bottom centered step pill

## Implementation notes

Use this design as a visual target. It is acceptable to adapt spacing and layout for VS Code webview constraints, but the tone should remain dark, premium, concise, and code-centered.
